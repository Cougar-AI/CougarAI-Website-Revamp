================================================================================

** REGISTER USER **
POST /auth/register
Content-Type: application/json

{
  "email": "test@university.edu",
  "password": "SecurePass123!",
  "first_name": "John",
  "last_name": "Doe"
}

Expected Response: 201
{
  "user_id": 1,
  "email": "test@university.edu",
  "message": "User registered successfully"
}

---

================================================================================
                            TESTING NOTES
================================================================================

1. AUTHENTICATION FLOW:
   - Register first with email/password
   - Login to get access_token and refresh_token
   - Use access_token in "Authorization: Bearer {token}" header for protected routes
   - Tokens expire, use refresh endpoint to get new ones

2. ERROR RESPONSES:
   - 400: Bad Request (missing/invalid parameters)
   - 401: Unauthorized (missing/invalid token)
   - 403: Forbidden (insufficient permissions)
   - 404: Not Found (resource doesn't exist)
   - 500: Internal Server Error

3. COMMON 404 ERRORS:
   - Student ID doesn't exist in profile table
   - Announcement/Event/Officer IDs don't match
   - Discord account not linked

4. TESTING TOOLS:
   - cURL: curl -X GET http://127.0.0.1:5000/api/profiles
   - Postman: Import endpoints and organize into folders
   - VS Code REST Client: Use .http files for quick testing
   - Thunder Client: VS Code extension for API testing

5. PAYMENTS NOTE:
   - Only allowed price IDs work (server-side validation)
   - Test IDs: price_1RPA0wQdq5f9y5dILdnU8jkY, price_1RPA1MQdq5f9y5dIX6qzElLY
   - Live IDs: price_1S4sVLH2XIQuLIalBvif5rrs, price_1S0ylVH2XIQuLIalbpMXxrV9

6. DATABASE CONSTRAINTS:
   - student_id: VARCHAR(50), PRIMARY KEY
   - user_id: INTEGER, UNIQUE per profile
   - email: UNIQUE
   - All timestamps are UTC
