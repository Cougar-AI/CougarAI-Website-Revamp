# Registration and Email Verification API

This document describes the newly implemented registration and email verification endpoints for the CougarAI website backend.

## Environment Variables

The following environment variables should be configured:

```bash
# Required for production
SECRET_KEY_BASE=your-secret-key
JWT_EMAIL_SECRET=your-jwt-email-secret
FRONTEND_URL=https://your-frontend-domain.com

# Email configuration
MAILER_BACKEND=smtp  # or 'disabled' for development
SMTP_HOST=smtp.your-provider.com
SMTP_PORT=587
SMTP_USER=your-email@domain.com
SMTP_PASSWORD=your-email-password
SMTP_USE_TLS=true
```

## API Endpoints

### POST /auth/register

Register a new user with email and password.

**Request:**
```json
{
  "email": "user@example.com",
  "password": "StrongPass123!"
}
```

**Response (201 Created):**
```json
{
  "ok": true
}
```

**Response (422 Validation Error):**
```json
{
  "field_errors": {
    "email": "Enter a valid email address",
    "password": "Password must be at least 8 characters long; Password must include at least one symbol"
  }
}
```

**Password Requirements:**
- Minimum 8 characters
- At least one uppercase letter
- At least one lowercase letter
- At least one digit
- At least one symbol
- Not in common password denylist

**Features:**
- Case-insensitive email handling (using PostgreSQL citext)
- Idempotent: duplicate registrations return success
- Automatic verification email sending
- Secure bcrypt password hashing

### POST /auth/verify-email

Verify user's email address using JWT token from email.

**Request:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Response (200 OK):**
```json
{
  "ok": true
}
```

**Response (401 Unauthorized):**
```json
{
  "error": "Invalid or expired token"
}
```

**Response (400 Bad Request):**
```json
{
  "error": "Token is required"
}
```

**Features:**
- JWT token validation with 24-hour expiration
- Idempotent: multiple verifications of same email return success
- Updates `email_verified_at` timestamp in database

## Database Schema

```sql
CREATE EXTENSION IF NOT EXISTS citext;

CREATE TABLE users (
    user_id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email             CITEXT NOT NULL UNIQUE,
    password_hash     TEXT,
    email_verified_at TIMESTAMP WITH TIME ZONE,
    last_login        TIMESTAMP WITH TIME ZONE,
    joined_at         TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL,
    is_active         BOOLEAN DEFAULT TRUE NOT NULL
);
```

## Example Usage

```bash
# Register a new user
curl -X POST http://localhost:5000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "SecurePass123!"}'

# Verify email (token from email)
curl -X POST http://localhost:5000/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}'
```
